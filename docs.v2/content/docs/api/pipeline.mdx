title: Pipeline
description: API reference for PipelineBehavior, sync and async.

Pipeline behaviors wrap request processing with middleware-style logic. The mediator discovers registered behaviors at dispatch and chains them around the handler for you. Sync (`pymediate`) and async (`pymediate.aio`) variants exist.

## PipelineBehavior

<Tabs items={['Sync', 'Async']}>
  <Tab value="Sync">
    ```python
    from collections.abc import Callable
    from typing import Any
    from pymediate import PipelineBehavior, Request

    class PipelineBehavior[RequestT: Request[Any]](ABC):
        @classmethod
        def should_apply(cls, request: Request[Any]) -> bool:
            """Determine if this behavior should apply to the given request."""

        @abstractmethod
        def __call__(self, request: RequestT, next: Callable[[], Any]) -> Any:
            """Execute the behavior's logic and call next to continue the pipeline."""
    ```
  </Tab>
  <Tab value="Async">
    ```python
    from collections.abc import Awaitable, Callable
    from typing import Any
    from pymediate.aio import PipelineBehavior

    class PipelineBehavior[RequestT: Request[Any]](ABC):
        @classmethod
        def should_apply(cls, request: Request[Any]) -> bool: ...

        @abstractmethod
        async def __call__(self, request: RequestT, next: Callable[[], Awaitable[Any]]) -> Any:
            """Execute the behavior's async logic and await next to continue the pipeline."""
    ```
  </Tab>
</Tabs>

### Type parameters

| Parameter | Description |
| --- | --- |
| `RequestT` | Which requests this behavior wraps: `Request` for all, a specific request type, or a mixin |

### `__call__(request, next)`

The behavior's body. `next` is a callable representing the rest of the pipeline — remaining behaviors and, ultimately, the handler. Call it (or `await` it in the async variant) to continue; skip it to short-circuit.

| | |
| --- | --- |
| **Args** | `request` — the request being processed; `next` — continues the pipeline and returns the response |
| **Returns** | The response (usually the result of `next()`, possibly transformed) |

### `should_apply(request)` <span className="text-fd-muted-foreground text-sm">classmethod</span>

Decides whether the behavior wraps a given request. The default implementation is an `isinstance` check against `RequestT`. Override it for matching logic beyond types:

```python
class BusinessHoursBehavior(PipelineBehavior[Request]):
    @classmethod
    def should_apply(cls, request: Request) -> bool:
        return 9 <= datetime.now().hour < 17
```

## Running a behavior without a mediator

There is no public pipeline class — the chain the mediator builds is internal. To exercise a behavior in isolation (in a unit test, say), call it directly; `next` is just a zero-argument callable:

<Tabs items={['Sync', 'Async']}>
  <Tab value="Sync">
    ```python
    behavior = LoggingBehavior(logger)
    handler = GetUserHandler(database)
    request = GetUserRequest(user_id=123)

    response = behavior(request, lambda: handler(request))
    ```
  </Tab>
  <Tab value="Async">
    ```python
    behavior = LoggingBehavior(logger)
    handler = GetUserHandler(database)
    request = GetUserRequest(user_id=123)

    response = await behavior(request, lambda: handler(request))
    ```
  </Tab>
</Tabs>

For anything beyond a one-off, register the behaviors and let the mediator run them — ordering, selection via `should_apply`, and provider lifetimes all come from registration:

```python
services = Services()
services.add(LoggingBehavior(logger))     # first registered = outermost
services.add(ValidationBehavior())
services.add(TimingBehavior(metrics))     # last registered = innermost
services.add(GetUserHandler(database))

mediator = Mediator(services.provider())
response = mediator.send(GetUserRequest(user_id=123))
```
