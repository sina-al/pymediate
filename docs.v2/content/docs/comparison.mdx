Two questions come up whenever you evaluate a library like this: why not one of the alternatives, and what does the indirection cost? This page answers both with evidence you can check.

## Where the data comes from

In July 2026, we read the source of the six most popular mediator libraries in the Python ecosystem, ranked by GitHub stars and PyPI downloads — their dispatch code, their registries, and their type signatures. The findings are summarized in aggregate rather than by name — the goal is to describe common approaches, not to grade individual projects — but every row below comes from that survey.

## Feature by feature

| Capability | PyMediate | Typical alternatives |
| --- | --- | --- |
| Request-to-handler link | A type parameter: `Handler[CreateUser]`. Dispatch is keyed on the class itself. | Class-name strings, naming conventions, or a separate `bind(request, handler)` call that can pair the wrong two. |
| Response type at the call site | `send()` returns the `T` declared in `Request[T]` — inferred by mypy, autocompleted by your editor. | `object`, `Any`, or a generic that isn't bound to the request. |
| Handler signature validation | At class-definition time. A wrong parameter type, a missing return annotation, or a sync/async mixup raises at import. | None of the six validate signatures — mistakes surface at runtime, when a request is dispatched. |
| Duplicate handler registration | Raises immediately, pointing at the file and line of the first registration. | Silently overwrites the original handler, or silently ignores the new one. |
| Sync API | A first-class structural mirror of the async API — same classes, same semantics. | Almost always async-only; synchronous codebases must adopt an event loop. |
| Event publishing (one request, many handlers) | Not yet — planned in [issue #10](https://github.com/sina-al/pymediate/issues/10). | Five of six have it. Every implementation surveyed is type-erased or async-only, but they have it today. |
| Streaming responses | Not yet — planned in [issue #12](https://github.com/sina-al/pymediate/issues/12). | One of the six offers it, untyped. |
| Runtime dependencies | Zero. Ships `py.typed` and is checked under `mypy --strict`; `dependency-injector` is an optional extra. | Varies — several require a serialization framework or a dependency-injection container to participate at all. |

Two of those rows favor the alternatives. If you need event publishing or streaming today, another library gets you there sooner. Both are on the [roadmap](https://github.com/sina-al/pymediate/issues), and when they land they'll be typed, consistent with the rest of the API.

## What does `send()` cost?

Indirection is never free — a direct call is always faster than a dispatched one. The honest questions are how much, and whether it matters. The benchmark measures PyMediate against the direct call it replaces, never against other libraries, because overhead numbers across different feature sets aren't comparable — the goal is a number you can reproduce on your own hardware.

The following results come from a 2020 Intel MacBook Pro running Python 3.13.0 (CPython) on macOS 15.7. Each value is the median of five samples of 100,000 calls, with a trivial handler so dispatch dominates.

| Scenario | Median per call | Relative to direct call |
| --- | ---: | ---: |
| Sync: `handler(request)` — direct call | 0.34 µs | Baseline |
| Sync: `mediator.send(request)` | 1.8 µs | 5.5x |
| Sync: `send()` plus one pipeline behavior | 8.3 µs | 24.8x |
| Async: `await handler(request)` — direct call | 0.42 µs | Baseline |
| Async: `await mediator.send(request)` | 2.1 µs | 5.0x |

In practice, routing through the mediator costs about 1.5 microseconds over calling the handler yourself, and a pipeline behavior adds a few microseconds more. For scale, a single indexed database query costs hundreds of microseconds and a network round trip costs milliseconds — if your handler does any real work, dispatch overhead disappears into the noise. If you dispatch millions of trivial in-memory operations per second on a hot path, a mediator — any mediator — is the wrong tool, and you should skip this library too.

Absolute numbers vary by machine. Treat the table as orders of magnitude, and rerun it yourself. The [benchmark script](https://github.com/sina-al/pymediate/blob/main/scripts/benchmark.py) is a self-contained [uv script](https://docs.astral.sh/uv/guides/scripts/) served by this site — it resolves the latest PyMediate release (its only dependency; PyMediate itself has none) and prints the exact version it measured. As with any script, read it before you run it from the network.

```bash
uv run https://pymediate.sina-al.uk/benchmark.py
```

From a repo checkout, `uv run poe benchmark` runs the same script against the local source instead.

## The short version

The mediator pattern is well established in Python, and the surveyed libraries implement it capably. What none of them offer is the type system holding the whole thing together: the request-to-handler link, the response type at the call site, and wiring validation before the first request is ever sent. That's PyMediate's bet, and [Why PyMediate?](/docs#why-pymediate) covers what it buys you day to day.
