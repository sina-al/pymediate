8class Event: 

9 """Base class for events published to zero or more handlers. 

10 

11 Where a `Request[ResponseT]` is sent to exactly one handler and returns its 

12 typed response, an `Event` is published to every handler subscribed to its 

13 exact type - zero, one, or many - and returns nothing. Inherit from `Event` 

14 to make a class publishable via `Mediator.publish()`. 

15 

16 Events carry no response type parameter: publishing is fire-and-notify, so 

17 there is nothing to infer and handlers must be annotated to return `None`. 

18 

19 This class works seamlessly with dataclasses, regular classes, and any 

20 Python class structure. 

21 

22 Examples: 

23 Defining and publishing an event: 

24 ```python 

25 from dataclasses import dataclass 

26 from pymediate import Event, EventHandler, Mediator, Services 

27 

28 @dataclass 

29 class OrderPlaced(Event): 

30 order_id: int 

31 

32 class SendConfirmation(EventHandler[OrderPlaced]): 

33 def __call__(self, event: OrderPlaced) -> None: 

34 print(f"confirming order {event.order_id}") 

35 

36 services = Services() 

37 services.add(SendConfirmation()) 

38 mediator = Mediator(services.provider()) 

39 

40 mediator.publish(OrderPlaced(order_id=42)) 

41 ``` 

42 

43 Note: 

44 Publishing dispatches on the exact class of the event instance - a 

45 handler subscribed to a base event class does not receive derived 

46 events. This mirrors how requests dispatch to handlers. 

47 

48 See Also: 

49 - EventHandler: Base class for handlers that subscribe to an event. 

50 - Mediator.publish: Publishes an event to all its handlers. 

51 - Request: The one-handler, typed-response counterpart. 

52 """

55class EventHandler[EventT: Event](EventHandlerBaseMixin[EventT], ABC): 

56 """Abstract base class for synchronous event handlers. 

57 

58 Event handlers contain the logic that reacts to a published event. Unlike 

59 request handlers, any number of event handlers may subscribe to the same 

60 event type - `Mediator.publish()` invokes every one of them, in registration 

61 order. 

62 

63 The handler performs class-definition-time validation via __init_subclass__ 

64 to ensure: 

65 - The __call__ method exists and is properly implemented 

66 - The __call__ method is synchronous (not async) 

67 - The __call__ parameter annotates the exact declared event type 

68 (not a base class or union) 

69 - The __call__ return annotation is None - event handlers produce no response 

70 

71 This validation happens at class definition time (import time), catching 

72 errors early in the development cycle rather than at runtime. 

73 

74 Type Parameters: 

75 EventT: The type of event this handler subscribes to. Must inherit 

76 from Event; static type checkers enforce the bound, and PyMediate 

77 validates it at class definition time. 

78 

79 Examples: 

80 Two handlers subscribed to one event: 

81 ```python 

82 from dataclasses import dataclass 

83 from pymediate import Event, EventHandler, Mediator, Services 

84 

85 @dataclass 

86 class OrderPlaced(Event): 

87 order_id: int 

88 

89 class SendConfirmation(EventHandler[OrderPlaced]): 

90 def __call__(self, event: OrderPlaced) -> None: 

91 print(f"confirming order {event.order_id}") 

92 

93 class UpdateAnalytics(EventHandler[OrderPlaced]): 

94 def __call__(self, event: OrderPlaced) -> None: 

95 print(f"recording order {event.order_id}") 

96 

97 services = Services() 

98 services.add(SendConfirmation()).add(UpdateAnalytics()) 

99 mediator = Mediator(services.provider()) 

100 

101 mediator.publish(OrderPlaced(order_id=42)) 

102 # confirming order 42 

103 # recording order 42 

104 ``` 

105 

106 Handler with dependencies: 

107 ```python 

108 class SendConfirmation(EventHandler[OrderPlaced]): 

109 def __init__(self, mailer: Mailer): 

110 self.mailer = mailer 

111 

112 def __call__(self, event: OrderPlaced) -> None: 

113 self.mailer.send(f"order {event.order_id} confirmed") 

114 ``` 

115 

116 Note: 

117 For asynchronous event handlers, use `pymediate.aio.EventHandler` 

118 instead. Validation occurs at class definition time: if your __call__ 

119 signature doesn't match expectations, you'll get a clear error message 

120 when the module is imported, not when the event is published. 

121 

122 Raises: 

123 InvalidHandlerSignatureError: If the __call__ signature is invalid, 

124 including a return annotation other than None. 

125 InvalidEventTypeError: If the event type doesn't inherit from Event. 

126 

127 See Also: 

128 - Event: Base event class. 

129 - Mediator.publish: Publishes an event to all its handlers (sync version). 

130 - pymediate.aio.EventHandler: Async event handler variant. 

131 """ 

132

135 @abstractmethod 

136 def __call__(self, event: EventT) -> None: 

137 """Handle the published event. 

138 

139 This is an abstract method that must be implemented by all EventHandler 

140 subclasses, with the signature 

141 `def __call__(self, event: EventType) -> None: ...` 

142 

143 Args: 

144 event: The event to handle. 

145 

146 Note: 

147 The annotation must be the exact event class - a base class or 

148 union passes static checking (contravariance) but raises 

149 `InvalidHandlerSignatureError` at class definition. The return 

150 annotation must be `None`: publishing has no response, and any 

151 value a handler returns is discarded. This method must also be 

152 synchronous; for async event handlers, use 

153 `pymediate.aio.EventHandler`. 

154 """
