Service resolution in PyMediate has two halves: `Services`, a mutable collection for registering instances, and `ServiceProvider`, the read-only protocol the mediator resolves from. Because `ServiceProvider` is a [protocol](https://docs.python.org/3/library/typing.html#typing.Protocol), any conforming implementation works â€” including the optional `dependency-injector` integration.

## ServiceProvider (protocol)

```python
from pymediate import ServiceProvider

class ServiceProvider(Protocol):
    def get(self, service_type: type[ServiceT]) -> ServiceT:
        """Get the first registered instance of the exact type."""

    def get_all(self, service_type: type[ServiceT]) -> Sequence[ServiceT]:
        """Get all instances of the type, including subclasses."""

    def has(self, service_type: type) -> bool:
        """Check whether any instance of the exact type is registered."""

    def get_all_types(self) -> tuple[type, ...]:
        """Get every exact type that has at least one registered instance."""
```

| Method | Returns | Raises |
| --- | --- | --- |
| `get(service_type)` | First registered instance of the exact type | `ServiceNotFoundError` if none registered |
| `get_all(service_type)` | All instances of the type **including subclasses**, in registration order | â€” |
| `has(service_type)` | `True` if any instance of the exact type is registered | â€” |
| `get_all_types()` | Every exact type with at least one registered instance | â€” |

## Services

```python
from pymediate import Services

class Services:
    def add(self, instance: object) -> Services: ...
    def provider(self) -> ServiceProvider: ...
    def clear(self) -> None: ...
```

### `add(instance)`

Registers a service instance by its concrete type (`type(instance)`). Multiple instances of the same type â€” including the same instance twice â€” are all kept and returned in registration order by `get_all()`.

| | |
| --- | --- |
| **Args** | `instance` â€” the service instance to register; cannot be `None` |
| **Returns** | `self`, so calls chain: `services.add(a).add(b)` |
| **Raises** | `ValueError` if `instance` is `None` |

### `provider()`

Builds an immutable `ServiceProvider` **snapshot** of the currently registered services. Later `add()`/`clear()` calls don't affect providers already created:

```python
services = Services()
services.add(MyService())
provider = services.provider()

services.add(AnotherService())
provider.has(AnotherService)  # False â€” registered after the snapshot
```

### `clear()`

Removes all registered services. Existing providers are unaffected â€” they're immutable snapshots.

## DependencyInjectorServiceProvider

```python
from collections.abc import Mapping
from typing import Any

from dependency_injector import containers, providers

from pymediate.providers import DependencyInjectorServiceProvider

class DependencyInjectorServiceProvider:
    def __init__(
        self,
        container: containers.Container,
        *,
        provider_types: Mapping[providers.Provider[Any], type[Any]] | None = None,
    ) -> None:
        """Index services declared by a Dependency Injector container."""
```

A `ServiceProvider` backed by a [`dependency-injector`](https://python-dependency-injector.ets-labs.org/) container â€” requires the `di` extra (`pip install 'pymediate[di]'`). It indexes declared providers **by their output type**, not by their attribute name, then implements the same four protocol methods. Indexing does not call providers.

Pass a real Dependency Injector container â€” `DeclarativeContainer` and `DynamicContainer` are
supported. Lookalike objects that merely expose a `providers` mapping are not supported and are
rejected at construction time. Declared `providers.Container` children are indexed recursively in
declaration order; injected dependencies are not traversed.

```python
from dependency_injector import containers, providers
from pymediate import Mediator
from pymediate.providers import DependencyInjectorServiceProvider

class AppContainer(containers.DeclarativeContainer):
    database = providers.Singleton(Database)
    create_user = providers.Factory(CreateUserHandler, database=database)

mediator = Mediator(DependencyInjectorServiceProvider(AppContainer()))
```

<Callout type="info">
  Class-backed providers and concretely annotated factory functions are inferred without
  construction. Pass an opaque provider in `provider_types` when its output cannot be inferred.
  The [dependency injection guide](/docs/guide/dependency-injection) covers nested containers,
  explicit types, and composing the mediator through `providers.Self()`.
</Callout>

## ServiceNotFoundError

```python
class ServiceNotFoundError(Exception):
    def __init__(self, service_type: type, available_types: list[type]) -> None: ...
```

Raised by `get()` when no instance of the requested exact type is registered. The message lists the types that *are* available. Note this is a plain `Exception`, not a `PyMediateError` â€” see [errors](/docs/api/errors).
