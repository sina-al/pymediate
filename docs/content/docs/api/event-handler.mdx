`EventHandler` defines one subscriber for an [`Event`](/docs/api/event) type. Several handler
classes may subscribe to the same event.

<Tabs items={['Async', 'Sync']}>
  <Tab value="Async">
    ```python
    from abc import ABC, abstractmethod

    from pymediate import Event, EventHandler

    class EventHandler[EventT: Event](ABC):
        @abstractmethod
        async def __call__(self, event: EventT) -> None: ...
    ```
  </Tab>
  <Tab value="Sync">
    ```python
    from abc import ABC, abstractmethod

    from pymediate.sync import Event, EventHandler

    class EventHandler[EventT: Event](ABC):
        @abstractmethod
        def __call__(self, event: EventT) -> None: ...
    ```
  </Tab>
</Tabs>

| Type parameter | Meaning |
| --- | --- |
| `EventT` | The exact `Event` subclass handled by this class |

## Definition-time validation

When Python defines an event-handler subclass, PyMediate checks that:

- `EventT` is an `Event` subclass;
- the subclass defines `__call__` with exactly one parameter besides `self`;
- that parameter is annotated with the exact `EventT` class;
- the return annotation is `None`; and
- `__call__` is `async def` for `pymediate.EventHandler` and plain `def` for
  `pymediate.sync.EventHandler`.

Validation can raise `InvalidEventTypeError` or `InvalidHandlerSignatureError`. Valid handler
classes are recorded in class-definition order. Unlike request handlers, event handlers do not
have a one-handler-per-type restriction.

The asynchronous and synchronous handler classes write to the same process-wide subscription
list. Every handler for one exact event type must use one API; do not mix
`pymediate.EventHandler` and `pymediate.sync.EventHandler` for that event. The asynchronous
mediator expects awaitable handlers, while the synchronous mediator calls handlers directly.

## Example

```python
from pymediate import EventHandler


class RecordOrderMetric(EventHandler[OrderPlaced]):
    async def __call__(self, event: OrderPlaced) -> None:
        print(f"recorded order {event.order_id}")
```

Register a `RecordOrderMetric` instance with the service provider used by the mediator before
publishing `OrderPlaced`.

## Introspection method

`get_event_type()` returns the declared event class, or `None` on an unparameterized base.
