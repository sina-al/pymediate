```python
from pymediate import Event

class Event: ...
```

Base class for events — facts published to zero or more subscribers via `Mediator.publish()`. Events are `send()`'s fan-out counterpart: where a request is routed to exactly one handler and produces a response, an event is delivered to every [`EventHandler`](/docs/api/event-handler) subscribed to its exact type, and returns nothing. See the [events guide](/docs/guide/events) for semantics and patterns; this page is the API contract.

Unlike `Request[ResponseT]`, `Event` takes **no type parameter** — publishing has no response, so there is nothing to infer. Inheriting from `Event` is what makes a class accepted by `Mediator.publish()` and usable as an `EventHandler` type parameter. Works with dataclasses, regular classes, and any class structure.

## Usage

With dataclasses (recommended):

```python
from dataclasses import dataclass
from pymediate import Event

@dataclass
class OrderPlaced(Event):
    order_id: int
    customer_email: str
```

Like requests, events dispatch on the **exact class** of the published instance — a handler subscribed to a base event class does not receive derived events.

<Callout type="info" title="Name events in the past tense">
  An event states a fact — `OrderPlaced`, `UserRegistered` — while a request asks for work.
  A class you're tempted to name imperatively (`SendEmail`) is probably a request.
</Callout>
