---
title: Errors
description: API reference for PyMediate's exception hierarchy.
---

All PyMediate-specific exceptions subclass `PyMediateError`. They split into **definition-time** errors (raised at import, when a class is defined — fatal programming mistakes, not meant to be caught) and **dispatch-time** errors (raised while routing a request — catchable). See the [error handling guide](/docs/guide/error-handling#pymediates-own-errors) for how to treat each group.

## PyMediateError

```python
from pymediate import PyMediateError

class PyMediateError(Exception):
    def __init__(self, message: str, docs_path: str | None = None): ...
```

Base exception for all PyMediate errors. Carries an optional `docs_path` pointing to the relevant documentation page, which subclasses use to append a "learn more" link to their messages.

## Definition-time errors

Raised by `RequestHandler.__init_subclass__` validation the moment a class is defined — the failing `import` is the signal. Fix the code; don't catch these.

### InvalidHandlerSignatureError

```python
class InvalidHandlerSignatureError(PyMediateError):
    def __init__(self, handler_type: type, issue: str): ...
```

The handler's `__call__` has the wrong shape: missing type annotations, wrong parameter count, a parameter annotation that isn't *exactly* the request class declared in `RequestHandler[RequestT]` (a base class or union is rejected — see [the exact-annotation contract](/docs/guide/handlers#the-exact-annotation-contract)), or async/sync mismatch (a plain `def __call__` on the async `RequestHandler`, or vice versa).

### InvalidRequestTypeError

```python
class InvalidRequestTypeError(PyMediateError):
    def __init__(self, request_type: type): ...
```

A handler was parameterized on a type that doesn't inherit from `Request[ResponseType]`.

### InvalidEventTypeError

```python
class InvalidEventTypeError(PyMediateError):
    def __init__(self, event_type: type): ...
```

An event handler was parameterized on a type that doesn't inherit from `Event`. Because `EventHandler`'s type parameter is bound to `Event`, mypy and basedpyright flag this statically too — the runtime error is the backstop for unchecked code.

### ResponseTypeMismatchError

```python
class ResponseTypeMismatchError(PyMediateError):
    def __init__(self, handler_type: type, expected_type: type, actual_type: type): ...
```

The handler's return annotation disagrees with the response type its request declares. Note this is *runtime* (import-time) validation — mypy doesn't catch this particular mismatch. See [type safety](/docs/advanced/type-safety#runtime-checking-at-class-definition-time).

### HandlerAlreadyRegisteredError

```python
class HandlerAlreadyRegisteredError(PyMediateError):
    def __init__(
        self,
        request_type: type,
        existing_handler: type,
        new_handler: type,
        existing_location: str | None = None,
    ): ...
```

A second `RequestHandler` subclass was defined for a request type that already has one. PyMediate enforces one handler per request type, process-wide. The message includes both handler names and, when available, the file and line where the first was registered. Common in test suites — see [testing](/docs/advanced/testing#a-test-isolation-gotcha-the-handler-registry-is-global).

## Dispatch-time errors

Raised while a request is being routed. Both usually indicate a misconfigured deployment (something was never registered) rather than bad client input — at a web edge they map to a 500, not a 4xx.

### HandlerNotFoundError

```python
class HandlerNotFoundError(PyMediateError):
    def __init__(self, request_type: type, available_handlers: list[type] | None = None): ...
```

`mediator.send()` received a request whose type has no registered handler. The message lists the handlers that are available.

```python
from pymediate import HandlerNotFoundError

try:
    response = await mediator.send(GetUserRequest(user_id=1))
except HandlerNotFoundError:
    logger.error("RequestHandler missing from services — config bug")
    raise
```

### ServiceNotFoundError

```python
class ServiceNotFoundError(Exception):
    def __init__(self, service_type: type, available_types: list[type]) -> None: ...
```

A `ServiceProvider.get()` call couldn't resolve the requested type. Lives in `pymediate.service` and — unlike the others — subclasses plain `Exception`, not `PyMediateError`.
